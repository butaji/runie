### Create Chat Completion

Source: https://docs.rs/async-openai/0.34.0/src/async_openai/chat.rs.html

Use this function to create a model response for a given chat conversation. Ensure 'stream: false' in the request if not using the streaming endpoint. This method returns a single chat completion object.

```rust
pub async fn create(
    &self,
    request: CreateChatCompletionRequest,
) -> Result<CreateChatCompletionResponse, OpenAIError> {
    #[cfg(not(feature = "byot"))]
    {
        if request.stream.is_some() && request.stream.unwrap() {
            return Err(OpenAIError::InvalidArgument(
                "When stream is true, use Chat::create_stream".into(),
            ));
        }
    }
    self.client
        .post("/chat/completions", request, &self.request_options)
        .await
}
```

### Make Chat Completion Request

Source: https://docs.rs/async-openai/0.34.0/index.html

Shows how to create and send a chat completion request using the async-openai client. It utilizes a builder pattern for request parameters and awaits the asynchronous response.

```rust
use async_openai::{Client, types::responses::{CreateResponseArgs}};

// Create client
let client = Client::new();

// Create request using builder pattern
// Every request struct has companion builder struct with same name + Args suffix
let request = CreateResponseArgs::default()
    .model("gpt-5-mini")
    .input("tell me the recipe of pav bhaji")
    .max_output_tokens(512u32)
    .build()?;

// Call API
let response = client
    .responses()      // Get the API "group" (responses, images, etc.) from the client
    .create(request)  // Make the API call in that "group"
    .await?;

println!("{:?}", response.output_text());
```

### POST /chat/completions (Streaming)

Source: https://docs.rs/async-openai/0.34.0/src/async_openai/chat.rs.html

Creates a completion for the chat message with streaming enabled. Responses are sent as a stream of server-sent events.

```APIDOC
## POST /chat/completions (Streaming)

### Description
Creates a completion for the chat message. If set to true, the model response data will be streamed to the client as it is generated using server-sent events.

### Method
POST

### Endpoint
/chat/completions

### Parameters
#### Request Body
- **request** (CreateChatCompletionRequest) - Required - The request object for creating a chat completion, with `stream` set to `true`.

### Request Example
```json
{
  "model": "gpt-3.5-turbo",
  "messages": [
    {"role": "system", "content": "You are a helpful assistant."},
    {"role": "user", "content": "Tell me a joke."}
  ],
  "stream": true
}
```

### Response
#### Success Response (200)
- **response** (ChatCompletionResponseStream) - A parsed SSE stream of chat completion chunk objects until a [DONE] message is received.

#### Response Example
```json
{
  "choices": [
    {
      "delta": {
        "role": "assistant"
      },
      "index": 0,
      "finish_reason": null
    }
  ]
}
```

```json
{
  "choices": [
    {
      "delta": {
        "content": "Why did the"
      },
      "index": 0,
      "finish_reason": null
    }
  ]
}
```

```json
{
  "choices": [
    {
      "delta": {
        "content": " scarecrow win an award?"
      },
      "index": 0,
      "finish_reason": null
    }
  ]
}
```

```json
{
  "choices": [
    {
      "delta": {},
      "index": 0,

    }

### Create Chat Completion Request

Source: https://docs.rs/async-openai/0.34.0/src/async_openai/types/chat/chat_.rs.html

Defines the structure for sending a chat completion request to the OpenAI API.

```APIDOC
## POST /chat/completions

### Description
Creates a model response for the given chat conversation.

### Method
POST

### Endpoint
/chat/completions

### Parameters
#### Request Body
- **messages** (Vec<ChatCompletionRequestMessage>) - Required - A list of messages comprising the conversation so far.
- **model** (String) - Required - Model ID used to generate the response.
- **modalities** (Option<Vec<ResponseModalities>>) - Optional - Output types that you would like the model to generate (e.g., ["text"], ["text", "audio"]).
- **verbosity** (Option<Verbosity>) - Optional - Constrains the verbosity of the model's response (low, medium, high).
- **reasoning_effort** (Option<ReasoningEffort>) - Optional - Constrains effort on reasoning for reasoning models (minimal, low, medium, high).
- **max_completion_tokens** (Option<u32>) - Optional - An upper bound for the number of tokens that can be generated.
- **frequency_penalty** (Option<f32>) - Optional - Number between -2.0 and 2.0. Penalizes new tokens based on existing frequency.
- **presence_penalty** (Option<f32>) - Optional - Number between -2.0 and 2.0. Penalizes new tokens based on whether they appear in the text so far.
- **web_search_options** (Option<WebSearchOptions>) - Optional - Configuration for web search tool usage.
- **top_logprobs** (Option<u8>) - Optional - Integer between 0 and 20 specifying the number of most likely tokens to return.
```

### async_openai/chat.rs

Source: https://docs.rs/async-openai/0.34.0/src/async_openai/chat.rs.html

The `create` function generates a chat completion response. It requires a `CreateChatCompletionRequest` and returns a `CreateChatCompletionResponse`. If the `stream` option is set to true in the request, this function will return an error, and `create_stream` should be used instead.
