### Basic Argument Example

Source: https://github.com/clap-rs/clap/blob/master/examples/git-derive.md

Demonstrates how to define and use a basic argument like 'PATH' for the 'add' subcommand. This shows how to capture user input for file paths.

```console
$ git-derive add
? failed
adds things

Usage: git-derive[EXE] add <PATH>...

Arguments:
  <PATH>...  Stuff to add

Options:
  -h, --help  Print help

$ git-derive add Cargo.toml Cargo.lock
Adding ["Cargo.toml", "Cargo.lock"]

```

### Subcommand Handler Structure with Enum Dispatch

Source: https://github.com/clap-rs/clap/blob/master/examples/git-derive.rs

Demonstrates organizing subcommands using derive macros with enum-based dispatch pattern. Shows how to separate command definitions from handlers via pattern matching.

```rust
#[derive(Debug, Parser)]
struct Cli {
    #[command(subcommand)]
    command: Commands,
}

#[derive(Debug, Subcommand)]
enum Commands {
    Clone { remote: String },
    Diff { base: Option<OsString>, head: Option<OsString>, path: Option<OsString>, color: ColorWhen },
}

fn main() {
    let args = Cli::parse();
    match args.command {
        Commands::Clone { remote } => println!("Cloning {remote}"),
        Commands::Diff { .. } => println!("Diffing"),
    }
}
```
