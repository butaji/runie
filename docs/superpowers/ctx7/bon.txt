### Deriving Traits on the Builder

Source: https://context7.com/elastio/bon/llms.txt

Use `#[builder(derive(Debug, Clone))]` to apply standard derives like `Debug` and `Clone` directly to the generated builder struct.

```rust
use bon::Builder;

#[derive(Builder)]
#[builder(derive(Debug, Clone))]
struct Query {
    table: String,
    limit: Option<u32>,
}

let builder = Query::builder().table("users".to_owned());

// Inspect the partially-built state
assert_eq!(format!("{builder:?}"), r#"QueryBuilder { table: \"users\" }"#);

// Clone the builder and finish independently
let b2 = builder.clone().limit(10).build();
let b3 = builder.build();

assert_eq!(b2.limit, Some(10));
assert_eq!(b3.limit, None);
```

### Deriving Clone, Debug, and Into for a Struct Builder

Source: https://github.com/elastio/bon/blob/master/website/src/reference/builder/top-level/derive.md

Use `#[builder(derive(Clone, Debug, Into))]` on a struct to automatically implement these derives for its builder. This allows cloning the builder, debugging its state, and converting it directly into the struct instance using `.into()`.

```rust
use bon::Builder;

#[derive(Builder)]
#[builder(derive(Clone, Debug, Into))] // [!code highlight]
struct Example {
    name: String,
    is_admin: bool,
    level: Option<u32>,
}

let builder = Example::builder().name("Bon".to_owned());

// We can clone the builder    // [!code highlight]
let builder = builder.clone(); // [!code highlight]

// We can debug-format the builder          // [!code highlight]
let builder_debug = format!("{builder:?}"); // [!code highlight]

assert_eq!(
    builder_debug,
    // Only the fields that were set will be output
    r#"ExampleBuilder { name: \"Bon\" }"#
);

let example: Example = builder
    .is_admin(true)
    // We can use `From<ExampleBuilder> for Example` instead of
    // calling .build() thanks to `Into` derive
    .into();
```

### Derive Debug for Struct Builder

Source: https://github.com/elastio/bon/blob/master/website/src/guide/basics/derives-for-builders.md

Use `#[builder(derive(Debug))]` on a struct to enable the `Debug` trait for its generated builder. This allows inspecting the builder's state during debugging. Only set fields are included in the debug output.

```rust
#[derive(bon::Builder)]
#[builder(derive(Debug))] // [!code highlight]
struct Example {
    name: String,
    is_admin: bool,
    level: Option<u32>,
}

let builder = Example::builder().name("Bon".to_owned());

// This will output the current state of the builder to `stderr`
dbg!(&builder);

// You can also format the debug output to `String`:
assert_eq!(
    format!("{builder:?}"),
    // Only the fields that were set will be output
    r#"ExampleBuilder { name: \"Bon\" }"#
);

// Finish building
builder.is_admin(true).build();
```

### Derive `Clone` and `Debug` for Bon Builder

Source: https://github.com/elastio/bon/blob/master/website/src/blog/bon-builder-v2-2-release.md

Add `#[builder(derive(Clone, Debug))]` to generate `Clone` and `Debug` implementations for the builder. This allows reusing partial builders by cloning and inspecting builder state for debugging.

```rust
use bon::Builder;

#[derive(Builder)]
#[builder(derive(Clone, Debug))] // [!code highlight]
struct Example {
    name: String,
    level: u32,
}

let builder = Example::builder()
    .name("Bon".to_owned());

// We can get the debug output of the builder
assert_eq!(
    format!("{builder:?}"),
    r#"ExampleBuilder { name: \"Bon\" }"#
);

let _ = builder
    // We can clone the builder
    .clone()
    .level(99)
    .build();

// Because we cloned the builder, it's still available here
let _ = builder
    .level(100)
    .build();
```

### `#[builder(derive(Debug, Clone))]` — Derives on the Builder

Source: https://context7.com/elastio/bon/llms.txt

You can derive standard traits like `Debug` and `Clone` directly on the generated builder struct using `#[builder(derive(Debug, Clone))]`. This allows you to inspect the builder's state or clone it for independent construction paths.
