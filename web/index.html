Runie — TypeScript to Rust Compiler with Zero Overhead

      
Runie is the developer experience layer for Rune, a compiler driver that makes .r.ts and .r.tsx files valid source for native Rust projects. It parses TypeScript-like syntax using SWC, validates a zero-overhead subset, and transpiles to Rust source code that compiles with rustc into static binaries with no runtime dependency.

      
What Runie Does

      
Write TypeScript and TSX with familiar syntax. Runie (powered by the Rune compiler) transpiles it to Rust with zero runtime overhead. Development builds use hot-reloadable cdylibs for live iteration. Release builds produce single static executables indistinguishable from hand-written Rust.

TypeScript to Rust Type Mapping

      
number maps to f64 or i32. string maps to String or &str. boolean maps to bool. T | null maps to Option<T>. Tagged unions map to Rust enums. Arrays map to Vec<T>. Result types replace exceptions.

      
Ownership Inference

      
Rune automatically infers Rust ownership: const bindings are immutable, let bindings are mutable. Function parameters that are only read become &T borrows. Parameters that are mutated become &mut T. Consumed values move. No manual lifetime annotations required.

      
Hot Reload Development

      
cargo rune dev watches .r.ts and .r.tsx files, transpiles to target/rune-cache/, compiles to a versioned dylib, and atomically swaps it while preserving AppState in the host binary. Release builds inline everything into a single static binary with LTO.

      
Agent-Friendly Compiler Design

      
Deterministic idempotent output. Structured JSON errors with auto-fix suggestions. One-pass-per-file architecture. Self-describing HIR. Example-driven rule engine. Test-as-spec integration suite. Full source maps from Rust errors back to .r.ts line numbers.
